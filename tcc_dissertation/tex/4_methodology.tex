\label{chap:methodology}
This chapter describes experimental designs and implementation details of this work. What libraries, hardware, datasets and procediments were used to achieve the results in chapter \ref{chap:results}. The goal of this study was to train and compare different \gls{DL} architectures to \gls{BC} images samples from \gls{CL} in order to detect \gls{CRC}.

\section{Datasets}
\label{sec_datasets}
For this survey, three different datasets were used, all of them suffered transformations to convert their respectives classes in \gls{BC} staintment.

\subsection{100,000 histological images of human colorectal cancer and healthy tissue}
\label{subsec_nct100k}
The main successfully \gls{DS} of this study was \gls{NCTCRCHE100K}, by Kather et al. \cite{kather2018crc100k}. Samples were collected from the NCT Biobank (National Center for Tumor Diseases, Heidelberg, Germany) and the UMM pathology archive (University Medical Center Mannheim, Mannheim, Germany). All images were 224 x 224 pixels at 0.5 MMP and classified as one of the nine classes above:
\begin{itemize}
    \item Adipose (ADI)
    \item ackground (BACK)
    \item debris (DEB)
    \item lymphocytes (LYM)
    \item mucus (MUC)
    \item smooth muscle (MUS)
    \item normal colon mucosa (NORM)
    \item cancer-associated stroma (STR)
    \item colorectal adenocarcinoma epithelium (TUM)
\end{itemize}

They already divided the \gls{DS} in three differents sets:
\begin{itemize}
    \item NCT-CRC-HE-100K: contains one hundred thousand images color-normalized using Macenko's method \cite{5193250}.
    \item CRC-VAL-HE-7K: contains 7180 image from 50 different patients with no overlap with the ones in "NCT-CRC-HE-100K". The classes of this dataset are:
    \item NCT-CRC-HE-100K-NONORM: created from the same raw data as "NCT-CRC-HE-100K", however with no color normalization applied.
\end{itemize}


\subsection{Collection of textures in colorectal cancer histology}
\label{subsec_crc5000}
\gls{CRC5000} by Kather et al. \cite{kather_2016_53169}, with two main folders:
\begin{itemize}
    \item Kather\_texture\_2016\_image\_tiles\_5000: Contains 5000 histological images of 150 x 150 px each, classified as one of the eight classes bellow:
    \begin{itemize}
        \item TUMOR
        \item STROMA
        \item COMPLEX
        \item LYMPHO
        \item DEBRIS
        \item MUCOSA
        \item ADIPOSE
        \item EMPTY
    \end{itemize}
    \item Kather\_texture\_2016\_larger\_images\_10: Contains 10 larger histological images of 5000 x 5000 px each, containing more than one tissue type.
\end{itemize}
 
\subsection{Lung and Colon Cancer Histopathological Image
Dataset (LC25000)}
\label{subsec_lc25000}
Composed of twenty five thousand images of colon and lung tissues, the \gls{LC25000} were created from originallt seven hundred fifty total images of lung tissue (250 benign lung tissue, 250 lung adenocarcinomas, and 250 lung squamous cell carcinomas) and 500 total images of colon tissue (250 benign colon tissue and 250 colon adenocarcinomas). All images were cropped to square sizes of 768 x 768 pixels from original 1024 x 768 pixels and augmented using the Augmentor software package from python programming language, by applying: left and right rotations (up to 25 degrees,1.0 probability) and by horizontal and vertical flips (0.5 probability).
The classes in the datasets are:
\begin{itemize}
    \item colon\_aca: colon adenocarcinomas
    \item colon\_n: benign colonic tissues
    \item lung\_aca: lung adenocarcinomas
    \item lung\_scc: lung squamous cell carcinomas
    \item lung\_n: benign lung tissues
\end{itemize}
All images were equally distributed between classes above, with five thousand samples for each one.

\section{Environment}
\label{sec_environment}
Considering hardware limitation by the lack of \gls{GPU} during experiments, it was necessary a cloud-based service to run the \gls{DL} models through training phase. Google Colab, a Jupyter Notebook environment provided by Google, was chosen due to facility in configuration and it`s reliability in the Pro+ plan. The service have a free use option, however it suffer from interrupting overtime and less capable \gls{GPU} accessible. All models ran with Nvidia A100-SXM4 with 40Gb VRAM.

All models Tensors were gotten from Pytorch python library. The ones used were:
\begin{itemize}
    \item AlexNet
    \item VGG 16
    \item ResNet 18
    \item ResNet 34
    \item ResNet 50
    \item ResNet 101
    \item ResNet 152
\end{itemize}

\section{Common Configurations to all scenarios}
\label{sec_config}

During train phase, all models listed in \ref{sec_environment} were read with batch size of 64 samples. With 50 epochs per train and 15 early stop behavior, all models were trained with the following optimizers configuration:
\begin{itemize}
    \item Adam: learning rate of $1\times 10^{-4}$.
    \item SGD: learning rate of $1\times 10^{-2}$, momentum of 0.9 and weight decay of $5\times 10^{-4}$.
    \item AdamMax: learning rate of $1\times 10^{-2}$, $\beta_1$ of 0.9, $\beta_2$ of 0.999, weight decay of $1\times 10^{-4}$ and $\varepsilon$ of $1\times 10^{-8}$.
\end{itemize}
All optimizers hyperparams were choosen based on references in chapter \ref{chap:related}, for a measurable comparasions with other results, as detailed in \ref{chap:results}. Also, when applied, scheduler was configured with step size of 5 and $\gamma$ of $0.1$.


\section{Procediments}
\label{sec_procediments}
All development was built in \gls{VSCODE}, an \gls{IDE} with the "Colab" extension, where all dependecies were easily installed with a Jupyter Notebook comand. The datasets were downloaded through their respective \gls{API} inside the respective notebook.

After downloading all images necessary, they were converted to a \gls{CSV} file per \gls{DS}. For all of them, the \gls{CSV} was generated with two columns: path and label. In the first one, was the path the program would need to course to get the image, while the second indicats the image class. For training using Pytorch library interface, label needed to be a numeric value, so the following transformation was made:
\begin{itemize}
    \item 1: cancer image
    \item 0: non-cancer image
\end{itemize}


With the respective \gls{CSV} file, the samples were splitted in validation, test and train, while rezised to 224 x 224 pixels and normalized using ImageNet standarts  - 0.485, 0.456, 0.406 mean and 0.229, 0.224, 0.225  standard deviation to aid training convergence and efficiency \cite{DBLP:conf/nips/KrizhevskySH12} \cite{10913097}.

After transformation, all images are put in memory instead of storage, through DataLoader for velocity enhancement during training in batches of 64.

Training is made using \gls{SL} with train and validation \gls{DS}. Through 50 epochs, each loss is calculated through Cross Entropy Function. If validation loss does not improve in 15 epochs, early stopped is activated and the best model, with best performancing (lowest validation loss), is saved and used for tests with test \gls{DS}.

\section{Train and test with LC25000}
\label{sec_first_attempt}
Based on Md. Bipul Hossain and Mohamed Shaban paper \cite{DBLP:conf/aibthings/HossainS24}, the \gls{DS} choosen in the first attempt for training and test was the \gls{LC25000}. First, lung images were discarted, remaining a total of ten thousand colon labeled images.

Those images were splited using $train\_test\_split$ function from $sklearn$ python library. For the params of the function, it was passed $42$ to the $random_state$ params, which controls the shuffling applied to the data before applying the split. This number also was passed for reproducible output across multiple function calls. 

During split process, the \gls{LC25000} \gls{DS} was divided between train and test, eighty percentage of the images for training, and twenty percentage for test usage. Then, those train images were also splitted with $train\_test\_split$ function, using the same parameters above, for create the validation image set. With ten percentage of those set for validation, and 90 percentage for train.
\begin{equation}
    train = 72\% = 7200  images
\end{equation}
\begin{equation}
    test = 20\% = 2000  images
\end{equation}
\begin{equation}
    validation = 8\%  = 800  images
\end{equation}

After train and saving the best model, it was validated with the 2000 images test \gls{DS}.

As shown in \ref{chap:results}, some of the models did not converge into the best model during train with \gls{LC25000}. Because of that, this procediments were repeated, but with the use of a scheduler with the configurations described in \ref{sec_config}.

\section{Train with LC25000 and test with CRC5000}
\label{sec_second_attempt}
The goal of this section was to verify if \gls{LC25000} was a reliable \gls{DS} for \gls{DL} applications, as discussed in the results \ref{chap:results} of \ref{sec_first_attempt}. Considering that, the only models used were AlexNet and Residual Network 152.

In this case, images from \gls{LC25000} were modified as in the section above, with the exception of not splitting for test, only train and validation.

For test, images of \gls{CRC5000} were converted in csv described in \ref{sec_config}, but excluding all images from the $03\_COMPLEX$ class, considering it could harm the system performance.
The best model was used to test with the 4375 images from remaining \gls{CRC5000} \gls{DS}.
